## Environment URL

Each [app environment](/13.objects/2.app-environment.md) has a unique public environment URL assigned that is constructed like `{{app-name}}-{{environment-id}}.frbit.app`. Use it during development, testing or [multi staging](/14.tips/multi-staging.md). Look up the actual name for your environment URL with the [dashboard](/14.concepts/dashboard.md) under the app environment overview.

## External domain

fortrabbit does not provide domain registration services. An "external domain" is a domain registered with a third party domain name registrar. You can connect an external domain with an app environment.

## Connect an external domain

First, add a new domain with the fortrabbit dashboard. You will need to choose an app environment as target and some set some additional configuration.

:DashboardLink{title="Add a new domain" path="/new/domain"}

At the end of the process you will be shown DNS entries to be configured for the domain. Login to your domain provider and populate the DNS settings as provided by fortrabbit. It can take a while until the new DNS settings will propagated. See our [DNS troubleshooting guide](/14.tips/dns-troubleshooting.md) if it is stuck.

## Using HTTPS via Let's Encrypt

Domains on fortrabbit can be accessed via `HTTP` and `HTTPS`. Please see the [HTTPS article](/14.tips/https-troubleshooting.md) for information on secured connections and SSL certificates.

## Root path settings

A root path is a folder in the [file system](/14.concepts/directories.md) of the [app environment](/13.objects/2.app-environment.md) where a domain will end. The root path is saved with the app environment for all it's domains. That means, no interface is offered to configure individual root paths per domain. You can however configure custom routing of domains in your code and config, either by .htaccess rules or by some routing options with your software.

## Multiple websites in your app environment

You probably want to host more than one website on your App like so:

1. `your-first-domain.com` resolves to `htdocs/your-first-domain/public`
2. `your-other-domain.com` resolves to `htdocs/your-other-domain/public`

We recommend not to do that, see our [one website per app article](/14.concepts/one-website-per-app.md).

## Wildcard domains

Wildcard domains are currently not supported at fortrabbit. Please file a support ticket to describe your use case.

## Configuring your domain for e-mail

So far we have covered how to route a domain to fortrabbit. To receive and send e-mails from your domain you will configure the MX record of your domain. Please see your e-mail hosting provider for instructions.
