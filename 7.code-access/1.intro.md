## As alternative to Git deployment

We usually recommend to use [Git deployment](/6.deployment/1.intro.md). But not every developer is used to [Git](/14.tips/git-intro.md) yet and it has steep learning curve. Also not all software is ready to be used with Git and [Composer](/14.tips/composer.md).

Use [rsync](/14.tips/rsync.md), to upload your website.

## As addition to Git deployment

## SSH

SSH (Secure Shell) is a network protocol to login to remote web servers. SSH is used in the terminal. It's pre-installed with most operating systems. With SSH you login to have an interactive session on the remote server - the [app environment](/10.objects/2.app-environment.md) in our case. SSH is not suited for transferring files from your computer to the server. See the [SSH guide](/7.code-access/3.ssh.md) for more details.

## SFTP

SFTP (Secure File Transfer Protocol) is a network protocol to transfer files to and from web servers. It's build on top of SSH. You will usually use it with a graphical interface. See the [SFTP guide](/7.code-access/4.sftp.md).

## SSH key authentication

To access code by either method, SSH keys are required. Upload your public SSH keys to your personal account with the dashboard. Read [more about SSH keys here](/14.tips/ssh-keys.md).
