## Access the database from your app environment

Usually there is a configuration file which is used from the App to connect to the fortrabbit database. This is what you need to fill in there:

- **Database Name**: `{{app-env-slug}}`
- **Database Username**: `{{app-env-slug}}`
- **Database Password**: `{{app-database-password}}` _[< how to recover](#obtain-the-mysql-password)_
- **Database Host**: `{{app-env-slug}}.mysql.{{region}}.frbit.com` _< not localhost_
- **Database Port**: `3306` (usually not required)

If possible, we recommend to use [MySQL env vars](/11.concepts/env-vars.md) instead of hard coding them into your configuration files. The automatically available environment variables are:

- `FORTRABBIT_DATABASE`: Database name
- `FORTRABBIT_DATABASE_USER`: Database username
- `FORTRABBIT_DATABASE_PASSWORD`: Database password
- `FORTRABBIT_DATABASE_HOST`: Database host
- `FORTRABBIT_DATABASE_PORT`: Database port

To give you an example on how to use the environment variables:

```php
$pdo = new \PDO(
  'mysql:host='. getenv('MYSQL_HOST'). ';dbname='. getenv('FORTRABBIT_DATABASE'),
  getenv('DATABASE_USER'),
  getenv('DATABASE_PASSWORD'),
);
$pdo->query("SELECT * FROM ...")
```

That is only a generic example. See our specific guides for: Laravel, Symfony, WordPress, Craft CMS which are often using zero-config style environment variables.
