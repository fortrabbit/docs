## Booking backups

Backups are an optional component available per environment in different plans.

## Scaling backups

The different plan sizes define how many backups will be created.

## Backup features

* File backups
* MySQL backups
* Low disk I/O
* No performance impacts during backup
* Stored off site, space does not account for [web storage](3.web-storage.md) usage
* End-to-end encrypted transport
* AES-256 encrypted storage
* Easy access from the dashboard

## Configuring backups

Backups are available to download from the dashboard.

- The web storage backup file is an archive of all the contents of the `htdocs` folder. 
- The MySQL storage files is a plain `sql` file, containing the result of a `mysqldump` of the database.

See the  [backup restore article](9.platform/4.details/backup-recovery.md) on how recover a website from the files included with the backups.
