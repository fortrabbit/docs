Autoscaling is an option that can be toggled per environment with the component booking.

| Autoscaling | Pricing        | Website availability                           |
| ----------- | -------------- | ---------------------------------------------- |
| **On**      | Not guaranteed | Will not go offline due to resource exhaustion |
| **Off**     | Guaranteed     | May go offline due to resource exhaustion      |

Autoscaling is available applies to the following components in the following ways:

| Component       | Autoscaling | Metric    |
| --------------- | ----------- | --------- |
| MySQL           | ✅ Yes      | Storage   |
| Traffic         | ✅ Yes      | Bandwidth |
| Web storage     | ✅ Yes      | Space     |
| PHP             | ❌ No       | -         |
| Key-value store | ❌ No       | -         |
| Jobs            | ❌ No       | -         |
| Backups         | ❌ No       | -         |

Components are grouped in tiers. Even with autoscaling on, price fluctuation are not likely.

## Details

- When autoscaling is on and a larger plan is selected, that plan will considered the minimum scaling
- For traffic the largest plan reached will applied for the full month
- Consider MySQL memory usage beside storage usage as well
