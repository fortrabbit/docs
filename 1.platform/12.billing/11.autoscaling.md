Autoscaling is an option that can be toggled per environment with the component booking.

| Autoscaling | Fixed price | Availability               |
| ----------- | ----------- | -------------------------- |
| **On**      | ‚ùå No       | Stays online               |
| **Off**     | ‚úÖ Yes      | Goes offline when exceeded |

Autoscaling applies to the following components in the following ways:

| Component                                                          | Autoscaling | Metric    |
| ------------------------------------------------------------------ | ----------- | --------- |
| [MySQL](/1.platform/09.components/02.mysql.md)                     | ‚úÖ Yes      | Storage   |
| [Traffic](/1.platform/09.components/04.traffic.md)                 | ‚úÖ Yes      | Bandwidth |
| [Web storage](/1.platform/09.components/03.storage.md)             | ‚úÖ Yes      | Space     |
| [PHP](/1.platform/09.components/01.php.md)                         | ‚ùå No       | -         |
| [Key-value store](/1.platform/09.components/08.key-value-store.md) | ‚ùå No       | -         |
| [Jobs](/1.platform/09.components/07.jobs.md)                       | ‚ùå No       | -         |
| [Backups](/1.platform/09.components/05.backups.md)                 | ‚ùå No       | -         |

Components are grouped in tiers. Even with autoscaling on, price fluctuation is not likely.

## Details

- Selected plan is the minimum when autoscaling is on
- For traffic, the largest plan reached applies for the month
- Monitor MySQL memory alongside database storage usage
