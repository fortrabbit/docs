When you login with [SFTP](/1.platform/04.code-access/04.sftp.md) or [SSH](/1.platform/04.code-access/03.ssh.md) to your [environment](/1.platform/10.objects/02.environment.md) you can see the file directory structure. This is what you'll find:

```raw
srv
  data
    www
    home
```

## www

The default web root (aka document root) directory is the main tree 'visible' from the web. You can change the routing [root path](/1.platform/06.settings/12.root-path.md), to any folder below the `htdocs` directory. The [git deployment](/1.platform/05.deployment/01.intro.md) syncs to the `htdocs` folder as well. `htdocs` is also your 'login folder' - starting point for SSH/SFTP. The whole path looks something like this:

- `/srv/data/www`

## home

Like your `~` on your local machine. Contains bash history, private SSH keys.

- `/srv/data/home`
