The new platform is fortrabbit rebuilt around a different model: a single Git workflow with [GitHub deployment](/1.platform/05.deployment/02.github-app.md), [components](/1.platform/03.concepts/03.components.md) you scale individually instead of fixed plans, and persistent storage that survives every deploy.

If you already host on the old platform, [new and old](/1.platform/01.new/4.new-and-old.md) summarises what changed and where to find an equivalent feature. If you are starting fresh, follow [first steps](/1.platform/02.get-started/01.first-steps.md) and skim the [BETA notes](/1.platform/01.new/3.beta.md) for the few things that are still in flight.
