Most websites performance problems can be backtraced to code and config issues. Most often it's oversight or missing experience and attention to [performance design](/3.dev/02.performance/03.best-practices.md) from the developer.

## Performance problem sources

Even with PHP based websites and web applications, PHP is not the only possible source of a slow website.

- Server side issues: PHP execution is slow
- Frontend issues: fat assets, blocking requests ‚Ä¶
- Config issues: Plugin misconfigured, expensive lookups
- Large datasets: Production setup has a ton of data
- Database problems: Slow queries,
- Hosting problems: bad neighbors, old hardware, cramped servers ‚Ä¶

## Refactor code

Only good code can fly. Review your code. Make it perform better. See our [performance design article](/3.dev/performance-design.md) for inspiration.

## Swap issues

In some cases swap usage by the application is not possible. In these cases you might see dreaded white-pages and along going error messages in the log: "Allowed memory size of 1234.. bytes exhausted (tried to allocate 234 bytes)." This should be understand as an urgent reminder to scale up the [PHP plan](/1.platform/09.components/1.php.md).
