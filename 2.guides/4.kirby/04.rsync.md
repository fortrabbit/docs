## Get ready

Before deploying Kirby with rsync, ensure you have these prerequisites installed and configured.

- Have a [local development environment](/4.integrations/01.local-development/01.intro.md) running.
- Have [Kirby installed locally](/2.guides/4.kirby/01.install.md).
- Have an account with fortrabbit.
- Have an app created at fortrabbit, ideally using the Kirby preset.

## Deployment

Synchronize your Kirby project files between your local machine and the fortrabbit environment using these rsync commands.

```shell
# UP: from local to remote (syncs to the app root)
$ rsync -av ./ {{app-env-id}}@ssh.{{region}}.frbit.app:

# DOWN: from remote to local (syncs from the app root)
$ rsync -av {{app-env-id}}@ssh.{{region}}.frbit.app: ./
```

See the [rsync article](/3.dev/20.how-to/rsync.md) for more details.

## The vendor folder

When syncing your project root with rsync, the Composer vendor folder is included; this works reliably in most cases. Ensure your local PHP version roughly matches the fortrabbit remote for best compatibility. Alternatively, you can run `composer install` on the fortrabbit environment, or use Git deployment with [build commands](/1.platform/05.deployment/03.build-commands.md) to automate dependency installation—see the [git and rsync workflow](/2.guides/4.kirby/03.git-and-rsync.md) guide for details.
