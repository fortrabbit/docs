## Get ready

Before installing Craft CMS locally, you need a working development environment and the official installation guide as reference.

- Have a [local development environment](/4.integrations/01.local-development/01.intro.md) running. [DDEV](/4.integrations/01.local-development/02.ddev.md) is recommended.
- Craft CMS 5 requires PHP 8.2 or newer, both locally and on fortrabbit.
- Use the [official Craft CMS install guide](https://craftcms.com/docs/5.x/install.html) as your guideline.
- Skip this step when you have a Craft project running locally and a GitHub repo connected [see below](#a---git-deployment-recommended).

## Install with Composer

Craft CMS 5 is installed with Composer. Run this command **on your local machine** to create a new project — the interactive setup wizard then starts automatically:

```shell
composer create-project craftcms/craft my-project
```

Using [DDEV](/4.integrations/01.local-development/02.ddev.md)? Prefix it: `ddev composer create-project craftcms/craft my-project`. See an error? Check your [local development](/4.integrations/01.local-development/01.intro.md) setup.

The deployment strategy is a separate choice made [further down](#choose-a-code-deployment-workflow): [Git](/1.platform/05.deployment/01.intro.md) for Composer-managed projects, or SFTP/rsync.

## Run the setup wizard

`composer create-project` launches the setup wizard automatically, asking for database and account details. The defaults mostly work and everything can be changed later.

To re-run it manually — for example after cloning an existing project — use the Craft console command from the project folder:

```shell
cd my-project
php craft setup
```

## Choose a code deployment workflow

Craft CMS projects consist of three parts requiring different deployment strategies: code (application files), database (content tables), and assets (media volumes). Decide now which deployment method suits your workflow.

- **Code** - Craft CMS (vendor) + templates/config _← decide now_
- [Database](/2.guides/3.craft-cms/04.database.md) - Content tables
- [Assets](/2.guides/3.craft-cms/05.assets.md) - Volumes

### A - Git deployment (recommended)

To connect your app environment to a GitHub repo set up one now. Init git and add GitHub as remote repo - either as a public or private. Here is an example using the GitHub CLI.

```bash
# Initialize a new git repository
git init

# Add files and commit
git add .
git commit -m "Init"

# Create a new public repository on GitHub from the current directory
gh repo create --public --source=. --push
```

We recommend to exclude asset volumes from Git to [sync them up and down](/2.guides/3.craft-cms/05.assets.md). In most cases this is already the case.

Skip this step if your code is already at GitHub, see [Git provider integration](/4.integrations/03.git-providers/01.intro.md) for other providers with CI/pipeline/actions flows. If this is fuzzy to you, see:

- [Deployment intro](/1.platform/05.deployment/01.intro.md)
- [GitHub CLI guide](/4.integrations/07.git-clients/02.gh-cli.md)
- [Git intro](/3.dev/15.learn/git-intro.md)
