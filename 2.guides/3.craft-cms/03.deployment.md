## Git deployment

Git deployment automatically updates your fortrabbit app when you push code to GitHub. If you've followed the Craft CMS guides so far, your local project is connected to a GitHub repo, and you've created an app on fortrabbit linked to that repo.

Trigger a deployment by running `git push` locally or clicking the deploy button in the dashboard. This pushes your latest commit to fortrabbit. See the [Git deployment intro](/1.platform/05.deployment/01.intro.md) for how this works. The fortrabbit platform also supports [other Git providers](/1.platform/04.code-access/01.intro.md).

## Deployment configuration

Craft CMS deployment is usually pre-configured automatically. fortrabbit's [stack detection](/1.platform/03.concepts/04.stack-detector.md) scans your repo and applies sensible defaults via [software templates](/1.platform/03.concepts/05.software-templates.md), so [build commands](/1.platform/05.deployment/03.build-commands.md) and [post-deploy commands](/1.platform/05.deployment/04.post-deploy-commands.md) are already set up in most cases.

### Deployment strategy

By default, Craft CMS apps use the [merge deployment strategy](/1.platform/05.deployment/08.strategy.md#merge-strategy) with [replace patterns](/1.platform/05.deployment/15.replace-patterns.md) for config files. This protects local [Craft asset volumes](/2.guides/3.craft-cms/05.assets.md) since they can be stored anywhere in your file structure.

Consider using the [replace strategy](/1.platform/05.deployment/08.strategy.md#replace-strategy) with [exclude patterns](/1.platform/05.deployment/10.exclude-patterns.md) for each asset volume. It works either way.

### Build commands

Build commands are usually pre-configured and run automatically during deployment:

```shell
composer install # Always
npm run prod # If detected
```

### Post deploy commands

Post-deploy commands run after your code is live on fortrabbit. These are usually pre-configured and run database migrations to apply schema changes and project config updates:

```shell
php craft project-config/apply
php craft migrate/all
```
